# دليل مشروع Pinsora (بينسورا) - التوثيق الكامل والمحدث 🚀

هذا الملف هو المرجع الأساسي والشامل لهيكلية وبرمجة موقع **Pinsora**، وهو منصة تواصل اجتماعي متكاملة لمشاركة الصور مدعومة بالذكاء الاصطناعي.

---

## 1. نظرة عامة على الموقع (Vision)
**Pinsora** هو مزيج بين Pinterest وشبكة اجتماعية حديثة. يتيح للمكتشفين والمبدعين التفاعل من خلال الصور، وبناء مجتمعات عبر المتابعة والدردشة، مع تغذية مستمرة للمحتوى بواسطة "مصنع صور" يعمل بالذكاء الاصطناعي 24/7.

---

## 2. الهيكلية التقنية (Tech Stack)
*   **الإطار الأساسي:** [Next.js 15](https://nextjs.org/) (App Router).
*   **لغة البرمجة:** TypeScript.
*   **قاعدة البيانات:** PostgreSQL مستضافة على [Supabase](https://supabase.com/).
*   **وسيط قاعدة البيانات (ORM):** [Prisma](https://www.prisma.io/) (v7).
*   **التخزين السحابي:** [Cloudflare R2](https://www.cloudflare.com/products/r2/) (تخزين الصور الفعلي).
*   **الذكاء الاصطناعي:** [Leonardo.ai](https://leonardo.ai/) (نموذج FLUX Schnell).
*   **الأمان:** [TensorFlow.js](https://www.tensorflow.org/js) (لفحص المحتوى NSFW).
*   **الأتمتة:** [GitHub Actions](https://github.com/features/actions) (لتشغيل المهام المجدولة).

---

## 3. الأنظمة والخدمات المتقدمة

### A. مصنع الصور السحابي (Internal AI Factory)
*   **المسار:** `/api/cron/generate`
*   **الآلية:** بدلاً من سكربت خارجي، الموقع يولد صوره بنفسه. يتم استدعاء هذا الرابط السري بواسطة "روبوت GitHub" كل 5 دقائق.
*   **الذكاء:** يختار "بوت" عشوائي من بين 30 حساباً، ويختار فئة تحتاج صوراً، ثم يولد الصورة ويحفظها.
*   **المراقبة:** يمكن رؤية النتائج لحظياً عبر صفحة "AI Factory Logs" في لوحة الأدمن.

### B. نظام التواصل الاجتماعي (Social Engine)
*   **المتابعة (Following):** نظام ربط بين المستخدمين مع زر تفاعلي في البروفايل.
*   **الدردشة (Direct Messages):** نظام مراسلة خاص بالكامل مبني داخل الموقع، يدعم التحديث التلقائي (Polling) كل 4 ثوانٍ.
*   **الإشعارات:** جرس تنبيهات يغطي (الإعجابات، الحفظ، المتابعة، والرسائل الجديدة).

### C. نظام الحماية الذكي (AI NSFW Filter)
*   **المسار:** `src/app/upload/page.tsx`
*   **الآلية:** فحص الصور داخل متصفح المستخدم باستخدام ذكاء اصطناعي (Model) يمنع رفع أي محتوى جنسي أو غير لائق قبل وصوله للسيرفر.

### D. لوحة التحكم (Admin Dashboard)
*   **المميزات:**
    *   إحصائيات إجمالية (مستخدمين، صور، أوسمة).
    *   إدارة المستخدمين (حذف الحسابات المخالفة).
    *   إدارة المحتوى (حذف الصور).
    *   مراجعة البلاغات (Reports).
    *   سجلات المصنع (AI Logs).

---

## 4. بنية المجلدات الهامة
-   `src/app/api/cron`: يحتوي على محرك التوليد الآلي.
-   `src/app/messages`: يحتوي على واجهة الدردشة.
-   `src/app/admin`: لوحة التحكم الكاملة.
-   `.github/workflows`: ملف إعدادات "المنبه" الذي يشغل التوليد من GitHub.

---

## 5. متغيرات البيئة (Secrets)
يجب توفر هذه المفاتيح في Vercel وفي GitHub Secrets لضمان عمل المصنع:
- `DATABASE_URL`: رابط قاعدة البيانات.
- `AUTH_SECRET`: مفتاح تشفير الجلسات والرابط السري للمولد.
- `LEONARDO_API_KEY`: مفتاح توليد الصور.
- `R2_...`: إعدادات التخزين السحابي لـ Cloudflare.

## 6. أوامر التشغيل الأساسية

| الأمر | الوظيفة |
| :--- | :--- |
| `npx prisma db push` | مزامنة الجداول الجديدة مع Supabase (مهم عند إضافة ميزات) |
| `npm run db:seed` | إنشاء الفئات (179 فئة) وحسابات البوتات (30 حساب) |
| `vercel --prod` | رفع وتحديث الموقع المباشر |
| `git push origin main` | تحديث كود "المصنع" على GitHub |
