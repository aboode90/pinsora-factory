# دليل مشروع Pinsora (بينسورا) - التوثيق الكامل 🚀

هذا الملف هو المرجع الأساسي لهيكلية وبرمجة موقع **Pinsora**، وهو منصة تواصل اجتماعي لمشاركة الصور مدعومة بالذكاء الاصطناعي.

---

## 1. نظرة عامة على الموقع
**Pinsora** هو موقع شبيه بـ Pinterest، يتيح للمستخدمين اكتشاف الصور، حفظها في لوحات، متابعة المبدعين، والدردشة المباشرة. يتميز الموقع بنظام توليد صور تلقائي يعمل بالذكاء الاصطناعي لتغذية الموقع بمحتوى متجدد 24/7.

---

## 2. الهيكلية التقنية (Tech Stack)
*   **الإطار الأساسي:** [Next.js 15/16](https://nextjs.org/) (App Router).
*   **لغة البرمجة:** TypeScript.
*   **قاعدة البيانات:** [PostgreSQL](https://www.postgresql.org/) مستضافة على [Supabase](https://supabase.com/).
*   **وسيط قاعدة البيانات (ORM):** [Prisma](https://www.prisma.io/).
*   **نظام الهوية:** [Auth.js / NextAuth v5](https://authjs.dev/) (يدعم جوجل والبريد الإلكتروني).
*   **التخزين السحابي:** [Cloudflare R2](https://www.cloudflare.com/products/r2/) (لتخزين صور المستخدمين).
*   **الذكاء الاصطناعي:** [Leonardo.ai](https://leonardo.ai/) (نموذج FLUX Schnell).
*   **واجهة المستخدم:** Tailwind CSS 4 + Radix UI + Lucide Icons.

---

## 3. بنية المجلدات الرئيسية
-   `/src/app`: يحتوي على الصفحات (Pages) والواجهات البرمجية (APIs).
-   `/src/components`: يحتوي على مكونات الواجهة (أزرار، قوائم، بطاقات الصور).
-   `/src/lib`: يحتوي على ملفات الإعدادات المشتركة (الاتصال بقاعدة البيانات، إعدادات الحماية).
-   `/prisma`: يحتوي على "مخطط قاعدة البيانات" (Schema) وملفات التوزيع (Seed).
-   `/image-generator`: مجلد مستقل يحتوي على سيرفر البايثون المسؤول عن توليد الصور تلقائياً.

---

## 4. الأنظمة والخدمات الأساسية

### A. نظام الحماية الذكي (AI NSFW Filter)
*   **الوظيفة:** منع رفع أي صور تحتوي على عري أو محتوى جنسي.
*   **التقنية:** يستخدم `TensorFlow.js` ومكتبة `nsfwjs` للفحص داخل متصفح المستخدم قبل الرفع.
*   **المسار:** `src/app/upload/page.tsx`.

### B. نظام توليد الصور (AI Generator)
*   **الوظيفة:** سيرفر يعمل في الخلفية يولد صوراً لـ 179 فئة مختلفة بأسماء 30 حساباً وهمياً (Bots).
*   **الآلية:** يولد صورة كل دقيقة، يرفعها لـ R2، ثم يحفظها في قاعدة البيانات.
*   **الملف الرئيسي:** `image-generator/generator.py`.

### C. نظام التواصل الاجتماعي
*   **المتابعة (Following):** يسمح بربط المستخدمين ببعضهم.
*   **الدردشة (Chat):** نظام رسائل خاص (Direct Messages) يعمل بتقنية Polling (تحديث كل 4 ثوانٍ).
*   **الإشعارات:** جرس تنبيهات يخبرك بكل جديد (إعجاب، متابعة، رسالة، تعليق).

### D. لوحة التحكم (Admin Dashboard)
*   **الدخول:** متاح فقط للإيميل `abdullaalbder185@gmail.com`.
*   **المميزات:** رؤية الإحصائيات، حذف أي صورة، حذف أي مستخدم، ومراجعة البلاغات.

---

## 5. قاعدة البيانات (Prisma Models)
الجداول الرئيسية في قاعدة البيانات:
1.  **User:** بيانات المستخدمين والأدوار (Admin/User).
2.  **Image:** بيانات الصور (العنوان، الوصف، الرابط، المصور).
3.  **Category:** تصنيفات الموقع (179 تصنيف حالياً).
4.  **Follow:** علاقات المتابعة بين المستخدمين.
5.  **Conversation & Message:** نظام الدردشة والرسائل.
6.  **Notification:** تنبيهات المستخدمين.
7.  **Report:** بلاغات المحتوى المخالف.

---

## 6. أوامر التشغيل الهامة

| الأمر | الوظيفة |
| :--- | :--- |
| `npm run dev` | تشغيل الموقع للتطوير المحلي |
| `npx prisma db push` | مزامنة تغييرات قاعدة البيانات مع Supabase |
| `npm run db:seed` | إنشاء الفئات وحسابات البوتات الأساسية |
| `python image-generator/generator.py` | تشغيل سيرفر توليد الصور الذكي |
| `vercel --prod` | نشر الموقع وتحديثه على الإنترنت |

---

## 7. نصائح التطوير المستقبلي
*   **السرعة:** عند زيادة المستخدمين، يفضل استخدام WebSockets للدردشة بدلاً من Polling.
*   **التخزين:** دائماً راقب استهلاك Cloudflare R2 لضمان عدم تجاوز المساحة المجانية.
*   **الذكاء الاصطناعي:** يمكن تغيير موديل التوليد من Leonardo إلى OpenAI بسهولة عبر ملف `.env`.
